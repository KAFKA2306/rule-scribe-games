POINT B: Gemini API Call - No Error Handling
Location: app/core/gemini.py → generate_structured_json()
Key Issue: Following Crash-Driven Development (CDD) pattern, there are NO try-catch blocks.
This means ANY API failure will bubble up as 500.

Potential Failures:
1. resp.raise_for_status() → Crashes on 401/403/429 (invalid key, rate limit, quotas)
2. resp.json() KeyError if response structure differs from expected
3. json.loads() → JSONDecodeError if Gemini returns invalid JSON
4. Missing API key (None) → httpx sends invalid header

Code Pattern (from CDD rules in .claude/rules/style.md):
  "The usage of try-catch in business logic is FORBIDDEN"
  "Fail Fast, Fail Loud."
  → This is CORRECT for the architecture, BUT requires proper environment setup
