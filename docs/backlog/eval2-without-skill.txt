## PART 1: EXTRACT PDF CONTENT
================================================================================

### Step 1.1: Extract Text from PDF
You'll need to convert the PDF to plaintext. Choose one:

Option A - Using Python with pdfplumber:
```python
import pdfplumber

pdf_path = "/path/to/ticket-to-ride-rulebook.pdf"
with pdfplumber.open(pdf_path) as pdf:
    rules_text = ""
    for page in pdf.pages:
        rules_text += page.extract_text() + "\n"

print(rules_text)
```

Option B - Using CLI tool `pdftotext`:
```bash
pdftotext /path/to/ticket-to-ride-rulebook.pdf rules-output.txt
```

### Step 1.2: Organize Extracted Text
Structure the text into sections:
- **Setup Instructions**: Components, board setup, initial conditions
- **Gameplay Rules**: Turn structure, player actions, mechanics
- **Scoring**: Victory conditions, point calculation
- **Special Rules**: Advanced optional rules

Example structure:
```
SETUP INSTRUCTIONS:
- Components: 225 train cards, gameboard, destination tickets...
- Setup: Each player receives $45 in train cards...

GAMEPLAY:
- On your turn, perform ONE of these actions:
  1. Draw 2 train cards
  2. Claim a route
  3. Draw destination tickets
```

### Step 2.1: Understand the Database Schema

The database uses Pydantic models (app/models.py). Key fields:

```python
class GameDetail(BaseSchema):

    slug: str | None                 # "ticket-to-ride" (auto)

    rules_summary: str | None        # Rules overview (Japanese)
    min_players: int                 # 2
    max_players: int                 # 5
    play_time: int                   # Minutes (60)
    min_age: int                     # 8
    published_year: int              # 2004
    is_official: bool                # True (from official PDF)
    image_url: str | None            # Game cover image
    bgg_url: str | None              # BoardGameGeek link
    structured_data: StructuredData  # Keywords, mechanics, elements
```

### Step 2.2: Translate to Japanese (Critical Step)

You must create Japanese content. Options:

Option A - Use Gemini API (Integrated):
```python
from app.core.gemini import GeminiClient
import asyncio

async def translate_rules(english_rules: str) -> dict:
    gemini = GeminiClient()
    prompt = f"""
Translate this English board game rulebook to Japanese.
Create content for FIRST-TIME PLAYERS (Explain Like I'm 5).
Use plain Japanese (Desu/Masu form), avoid Katakana jargon.

ENGLISH RULES:
{english_rules}

Return JSON:
{{
    "title_ja": "Japanese title",
    "summary": "1-sentence Japanese summary",
    "description": "3-5 sentence Japanese description",
    "rules_content": "Detailed step-by-step Japanese rules"
}}
"""
    result = await gemini.generate_structured_json(prompt)
    return result

result = asyncio.run(translate_rules(extracted_rules_text))
```

Option B - Manual translation:
- Use DeepL API or human translator

### Step 2.3: Structure Rules Content (Markdown Format)

Follow this format (from prompts.py):

```markdown
[Game overview 2-3 sentences. No jargon. What experience?]

- [Component list]

1. [Setup steps numbered. Concrete and detailed]

[Detailed step-by-step instructions. Technical terms explained in plain Japanese]

[Clear end condition and victory rules]

- [3 strategy tips]
```

Example for Ticket to Ride:
```markdown
アメリカ大陸を列車で旅してネットワークを築くゲームです。
目的地チケットの目標地点を繋げて、効率よく得点を稼ぐかが勝負です。

- 列車カード 225枚（色別 各45枚）
- 目的地チケット 81枚
- プレイヤー駒（各色 45個）
- ゲームボード

1. 各プレイヤーが45ドル分の資金を受け取ります
2. 目的地チケットを3〜4枚ずつ配ります
3. プレイヤー駒を出発地に置きます

手番では以下のいずれかを行います：
1. 列車カードを2枚引く
2. 路線を占拠する（列車カードを支払って線を埋める）
3. 目的地チケットを引く

ゲーム終了：誰かが駒45個を使い切ったらラウンド終了。
最終得点：目的地チケット（達成で加点、未達成で減点）+ 占拠路線の得点。

- 短い路線から占拠する：早期に点数を稼げる
- 無理な目的地は避ける：減点が非常に痛い
- ライバルの動きを観察：狙いの路線を先制する
```

## PART 6: COMMON PITFALLS & SOLUTIONS
================================================================================

1. **Problem**: Japanese translation too technical (Katakana)
   **Solution**: Replace with plain Japanese
   - "ドラフト" → "カードを選ぶ"
   - "トリックテイキング" → "勝ったカードを取る"

2. **Problem**: PDF extraction produces garbled text
   **Solution**:
   - Try pdfplumber instead of PyPDF2
   - Manually clean text, remove OCR errors

3. **Problem**: Slug conflicts
   **Solution**: Add year/edition
   - "ticket-to-ride" → "ticket-to-ride-2004"

4. **Problem**: Database upsert fails validation
   **Solution**: Check required fields
   - title, slug, description, rules_content filled
   - Types correct (int for play_time)

5. **Problem**: Rules content wrong length
   **Solution**: Target 800-1500 Japanese characters
