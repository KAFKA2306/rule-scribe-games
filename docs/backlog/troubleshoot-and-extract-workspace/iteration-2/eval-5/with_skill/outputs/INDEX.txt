================================================================================
ROOT CAUSE SUMMARY (4 ISSUES)
================================================================================

Issue #1: Connection Pool Exhaustion
  50 games × 4 ops/game = 200 async tasks
  Python thread pool = 8 workers
  Supabase pool = 10-20 connections
  Result: Pool exhausted by game 15

Issue #2: Sync-to-Async Mismatch
  anyio.to_thread.run_sync() creates new thread per call
  Threads accumulate without reuse
  No backpressure or concurrency control

Issue #3: No Rate Limiting
  Batch script loops without delays
  Submits 4 requests per game instantly
  Overwhelms connection queue

Issue #4: Silent Exception Handling
  except Exception: pass hides real errors
  Violates crash-driven development
  Makes debugging impossible

Solution: Add semaphore to limit concurrency + batch processing + retry logic
