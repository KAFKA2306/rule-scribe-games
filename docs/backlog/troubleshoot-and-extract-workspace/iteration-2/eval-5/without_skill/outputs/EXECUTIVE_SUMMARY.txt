PROBLEM
-------
Batch PDF extraction script hangs ~game 15-20 when processing 50+ games.
Error: "connection pool exceeded"
Symptom: Manual kill required; script never completes naturally

ROOT CAUSE
----------
5 critical issues identified:

1. CRITICAL: Single synchronous Supabase client shared across unbounded
   concurrent threads. By game 15, all 10 connection slots occupied. Game 16+
   queue indefinitely.
   Location: app/core/supabase.py:7 (_client = create_client(...))

2. CRITICAL: Unbounded thread concurrency in batch processor.
   Each game spawns a thread requesting a connection; 50 games = 50 thread
   requests with no limiter. Semaphore to rescue.
   Location: batch_processor.py:46-48 (asyncio.gather without semaphore)

3. HIGH: No timeout on database queries. Default 60s blocks thread
   indefinitely if Supabase is slow, compounding pool starvation.
   Location: All supabase.py execute() calls

4. MEDIUM: Exponential backoff retry logic in application code violates
   Crash-Driven Development principle. Masks errors behind sleeps.
   Location: batch_processor.py:118 (2^attempt backoff)

5. LOW: Missing connection pool warmup on startup adds 1-2s latency to
   first batch game.

THE FIX (4 Code Changes, 11 Minutes Total)
--------------------------------------------

FIX #1 (CRITICAL - 5 minutes):
  File: app/core/supabase.py

  Add at module level:
    import asyncio
    _db_semaphore = asyncio.Semaphore(3)

  Wrap all async functions:
    async def search(...):
        async with _db_semaphore:
            def _q(): ...
            return await anyio.to_thread.run_sync(_q)

  Apply to: search(), upsert(), get_by_id(), get_by_slug(), list_recent(),
            increment_view_count() (7 functions total)

  Effect: Limits concurrent DB ops to 3; prevents thread pool explosion

FIX #2 (HIGH - 3 minutes):
  File: app/core/supabase.py

  Wrap each anyio.to_thread.run_sync() with asyncio timeout:
    try:
        async with asyncio.timeout(30):
            return await anyio.to_thread.run_sync(_q)
    except asyncio.TimeoutError:
        raise TimeoutError(f"Query exceeded 30s")

  Effect: Forces errors to surface; prevents indefinite hangs

FIX #3 (MEDIUM - 1 minute):
  File: Batch processing script

  Change:
    delay_between_batches=0.5  →  delay_between_batches=2.0

  Effect: Allows thread pool + connections to cool between batches

FIX #4 (MEDIUM - 2 minutes):
  File: batch_processor.py + Taskfile.yml

  Remove: batch_process_with_retry() function entirely
  Add to Taskfile:
    batch:extract-pdf:
      cmds:
        - until python scripts/batch_pdf_extract.py; do sleep 10; done

  Effect: Move retry logic to infrastructure (CDD compliant)
