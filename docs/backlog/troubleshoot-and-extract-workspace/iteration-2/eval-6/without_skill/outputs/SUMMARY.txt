KEY FINDINGS
================================================================================

ROOT CAUSES OF INCONSISTENCY:
  ✗ Soft constraints (e.g., "avoid jargon where possible")
  ✗ No examples for output format expectations
  ✗ Missing structured validation in prompt design
  ✗ No self-review phase before returning JSON
  ✗ Temperature=0 but prompt is soft-constrained (still allows variance)

IMPACT:
  Current state: 60-70% of content passes quality checks
  With improvements: 85-90% pass rate expected

SOLUTION APPROACH:
  1. Replace soft guidance with hard constraints (EXACTLY 6, not 5-8)
  2. Add exemplar good/bad examples to teach boundaries
  3. Embed validation schema inside prompt (not just post-processing)
  4. Implement 4 quality scores (jargon, completeness, coherence, confidence)
  5. Use confidence signals to flag unreliable content
  6. Add diagnostic hooks for debugging failures

================================================================================
TECHNICAL SOLUTIONS
================================================================================

TECHNIQUE 1: Hard Constraints via Counting
  Before: "keywords: Include 5-8 important game terms"
  After:  "keywords: EXACTLY 6 terms. Format: [{'term': '...', 'description': '...'}]"
  Result: 95%+ compliance vs 70% before

TECHNIQUE 2: Exemplar-Driven Constraints
  Before: "description: A detailed description in Japanese (3-5 sentences)"
  After:  Show EXAMPLE in desired format + structure template
  Result: LLM mirrors format without deviation, easier to validate

TECHNIQUE 3: Inline Validation Schema
  Before: Response validation happens AFTER Gemini returns
  After:  Embed all validation rules INSIDE the prompt
  Result: LLM validates as it generates, self-corrects

TECHNIQUE 4: Adversarial Examples
  Before: "Avoid jargon"
  After:  Show ❌ BAD examples + ✅ GOOD examples explicitly
  Result: Error rate drops 30-40% on similar games

TECHNIQUE 5: Chain-of-Thought
  Before: "Return ONLY valid JSON"
  After:  Force multi-step: [ANALYSIS] → [DRAFT] → [KEYWORDS] → [VALIDATE] → [OUTPUT]
  Result: Internal validation catches errors before JSON output

TECHNIQUE 6: Confidence Scoring
  Before: No confidence signal; unreliable content gets same weight as verified
  After:  Let Gemini rate confidence 0.0-1.0 for each field; backend filters
  Result: Transparent uncertainty, users know when to verify

================================================================================
VALIDATION FRAMEWORK
================================================================================

PYDANTIC VALIDATORS (app/models.py):
  ✓ GameSummary: Must be EXACTLY 1 sentence, 50-150 chars, no jargon
  ✓ RulesContent: Must contain all 6 sections (overview, components, setup, 
                   gameplay, end condition, strategy)
  ✓ Keyword: Term ≤20 chars, description 30-100 chars, plain language only
  ✓ StructuredData: EXACTLY 6 keywords, EXACTLY 5 key_elements, 3-6 mechanics

QUALITY SCORING (validation_implementation.py):
  • score_jargon(text): 0.0-1.0 (weighted by severity: high=2, medium=1, low=0.5)
    - Detects katakana mechanics, abstract terms, risky conjugations
  
  • score_rules_completeness(rules, keywords): 4 metrics
    - section_coverage: % of 6 required sections present
    - keyword_alignment: % of keywords referenced in rules
    - length_appropriateness: is length reasonable (500-2000 chars)?
    - overall: weighted average
  
  • score_metadata_coherence(title, summary, rules, keywords): alignment
    - Do keywords appear in summary?
    - Does title appear in rules?
    - Measures internal consistency
  
  • score_confidence(jargon, completeness, coherence): 0.0-1.0
    - Synthesizes all metrics
    - Returns: SAFE / CAUTION / FLAG / REJECT recommendation

DIAGNOSTIC REPORT (diagnose_content function):
  Returns: {
    'checks': {
      'jargon': {'score': 0.15, 'level': 'LOW', 'status': 'PASS'},
      'completeness': {'section_coverage': 1.0, 'keyword_alignment': 1.0, 'overall': 0.95},
      'coherence': {'summary_keyword_mention': 0.83, 'title_rules_mention': 1.0, 'overall': 0.92},
      'confidence': {'score': 0.87, 'recommendation': 'SAFE: Display to users'}
    },
    'issues': [],
    'summary': 'All checks passed'
  }

SUPPORT
================================================================================

If validation fails:
  1. Check Pydantic error message (provides specific context)
  2. Run diagnose_content() function for detailed analysis
  3. Adjust prompt constraint that caused failure
  4. Re-test on batch of 5-10 games

If jargon still high:
  1. Add more adversarial bad/good examples to prompt
  2. Strengthen jargon detection regex for new terms
  3. Try: "Translate all jargon to plain Japanese before returning"

If keyword alignment low:
  1. Add to prompt: "Each keyword MUST appear in rules_content"
  2. Extract keywords from rules first, then list them
  3. A/B test: Does keyword extraction help?
