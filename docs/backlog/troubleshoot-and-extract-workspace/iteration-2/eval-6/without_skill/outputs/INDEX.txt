FILE 3: validation_implementation.py (468 lines)
================================================================================
Production-ready Python code for validators and quality scoring.

Classes & Functions:

VALIDATORS:
  • GameSummary - Validates single-sentence summaries (50-150 chars, no jargon)
  • RulesContent - Ensures all 6 rule sections present
  • Keyword - Term validation (<=20 chars) + jargon detection in descriptions
  • JargonLevel - Enum for jargon severity (NONE/LOW/MEDIUM/HIGH)

SCORING FUNCTIONS:
  • score_jargon(text) → 0.0-1.0
    - Weighted detection (high terms=2, medium=1, low=0.5)
    - Normalizes to [0, 1] scale
  
  • score_rules_completeness(rules, keywords) → dict with 4 metrics
    - section_coverage: % of 6 sections present
    - keyword_alignment: % of keywords in rules
    - length_appropriateness: is length reasonable?
    - overall: weighted average
  
  • score_metadata_coherence(title, summary, rules, keywords) → dict
    - summary_keyword_mention: are keywords in summary?
    - title_rules_mention: is title in rules?
    - overall: weighted combined score
  
  • score_confidence(jargon, completeness, coherence) → 0.0-1.0
    - Synthesizes all metrics
    - Returns confidence score + recommendation

DIAGNOSTICS:
  • diagnose_content(title, summary, rules, keywords) → full report
    - Analyzes 4 dimensions (jargon, completeness, coherence, confidence)
    - Identifies issues and provides recommendations
    - Example output: {'title': '...', 'checks': {...}, 'issues': [...], 'summary': '...'}

HELPER FUNCTIONS:
  • _jargon_level() - Convert score to severity level
  • _confidence_recommendation() - Map confidence to action (SAFE/CAUTION/FLAG/REJECT)
  • _generate_summary() - Human-readable check results

EXAMPLE USAGE:
  Working Catan test case showing validation of complete metadata.

When to use: Copy into app/utils/quality_scorer.py; use in game_service.py
